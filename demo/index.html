Why scroll-magic-lite

Built for performance.

Every technical decision was made to keep animations butter-smooth on any device.

⚡

No scroll listeners

Uses the browser's native IntersectionObserver. No scroll events firing 60× per second.

🎯

GPU-only animations

Only transform and opacity are animated — the two properties that never trigger layout or paint.

♻️

Shared observer pool

All elements with the same threshold share one IntersectionObserver instance. 1 element or 1000 — same overhead.

🌐

Next.js / SSR safe

Every browser API call is guarded. No window is not defined errors in server components.

🏷️

Data-attribute API

Add data-sml="fadeUp" to any HTML element. No JS required. Works with any framework or plain HTML.

📦

Tree-shakeable ESM

Full ESM + CJS dual build. Import only what you use — bundlers will drop everything else.

Get started

3 ways to use it.

Pick the integration that fits your stack. 

<!-- 1. Add the script -->
<script src="https://unpkg.com/scroll-magic-lite/dist/scroll-magic-lite.global.js"></script>

<!-- 2. Add data-sml to any element -->
<h1 data-sml="fadeUp">Hello world</h1>
<p data-sml="fadeUp" data-sml-delay="150">Subtitle</p>
<div data-sml="zoomIn" data-sml-duration="800">Card</div>

<!-- That's it. Auto-initializes on DOMContentLoaded. -->
// npm install scroll-magic-lite
import { useScrollReveal } from 'scroll-magic-lite/react'

export default function Hero() {
  const titleRef = useScrollReveal({ animation: 'fadeUp', duration: 700 })
  const textRef  = useScrollReveal({ animation: 'fadeUp', delay: 150 })

  return (
    <section>
      <h1 ref={titleRef}>Hello world</h1>
      <p  ref={textRef}>Subtitle text</p>
    </section>
  )
}
// npm install scroll-magic-lite
import { ScrollMagicLite } from 'scroll-magic-lite'

const sml = new ScrollMagicLite()

// CSS selector — auto-staggers multiple matches
sml.reveal('.hero-title', { animation: 'fadeUp' })
sml.reveal('.card',       { animation: 'zoomIn', duration: 500 })

// Clean up (e.g. on route change)
sml.destroy()
